# Supported Operators

This document defines the operators our runtime (`onnx-runtime-ep-cpu`, `onnx-runtime-ep-cuda`) must implement.

**Scope:**
- ONNX standard ops: opset 17–26 (latest)
- Microsoft contrib ops (`com.microsoft` domain): subset used by Mobius-generated GenAI models

**Non-goal:** We don't implement the full ~180 ONNX op catalog. We target ops used by
transformer/LLM/diffusion/vision models generated by Mobius. For long-tail ops, legacy
plugin EPs (via C ABI bridge) handle them.

Implemented by the CPU session executor (not leaf EP kernels): a `Sequence`

## 2. Microsoft Contrib Ops (`com.microsoft` domain)

These are the ops Mobius/OGA generates in transformer models. They are **fused** ops
that combine multiple standard ops into a single optimized kernel.

### Tier 1: Critical (Mobius LLM output uses these directly)

| Op | Description | Fuses |
|----|-------------|-------|
| **GroupQueryAttention** | GQA with paged KV cache support | Q×K^T, scale, mask, softmax, ×V, with GQA head sharing. Supports paged block table input. |
| **MultiHeadAttention** | Standard MHA | Same as above but equal Q/K/V heads. |
| **RotaryEmbedding** | RoPE position encoding | cos/sin computation + rotation. Supports interleaved and non-interleaved. |
| **SkipLayerNormalization** | Residual + LayerNorm fused | Add(residual, input) → LayerNorm. One kernel, one memory pass. |
| **SkipSimplifiedLayerNormalization** | Residual + RMSNorm fused | Add(residual, input) → RMSNorm (no mean subtraction). |
| **SimplifiedLayerNormalization** | RMSNorm | Norm without mean subtraction (Llama-style). |
| **MatMulNBits** | INT4 quantized MatMul | Weight is INT4 packed. Dequant + GEMM fused. |
| **BiasAdd** | Add bias to tensor | Simple but fused for scheduling. |
| **FastGelu** | Approximated GELU | tanh approximation, faster than exact. |
| **BiasGelu** | Bias + GELU fused | MatMul output → add bias → GELU in one kernel. |
| **BiasSplitGelu** | Bias + Split + GELU (for GLU variants) | Used in SwiGLU/GeGLU architectures. |
| **QuickGelu** | x × sigmoid(α × x) | Used in some CLIP/Phi models. |
| **GatedRelativePositionBias** | Relative position bias with gating | |

| Op | Description | Notes |

## 3. Implementation Priority Matrix

```
Phase 1 (CPU EP — basic inference):
  Standard Tier 1 (core math + reshape + gather + layernorm + softmax)
  Contrib: SimplifiedLayerNormalization, SkipLayerNormalization, FastGelu, BiasAdd

Phase 2 (CUDA EP — LLM decode):
  Standard Tier 1 complete
  Contrib: GroupQueryAttention, RotaryEmbedding, MatMulNBits,
           SkipSimplifiedLayerNormalization, BiasGelu, BiasSplitGelu

Phase 3 (Full LLM support):
  Standard Tier 2 (quantization ops, einsum, trilu, range)
  Contrib: MultiHeadAttention, PagedAttention, SparseAttention,
           DecoderMaskedMultiHeadAttention, PackedAttention

Phase 4 (Multimodal + Diffusion):
  Standard Tier 2 complete (conv, resize, pool, batchnorm)
  Contrib: SkipGroupNorm, NhwcConv, FusedConv

Phase 5 (Full coverage):
  Standard Tier 3
  Contrib Tier 3
```

## 5. Minimum Opset Version Policy

We support **opset 17 as minimum** (introduced LayerNormalization as standard op).
Models with opset < 17 should be upgraded with `onnx.version_converter` before use.

For each op, we implement the **latest semantics** and provide backward-compat shims
for older opset versions where behavior differs (e.g. Reshape axis handling, Reduce* keepdims default).

**Kernel trait (every kernel implements this):**
```rust
pub trait Kernel: Send + Sync {
    /// Kernel name for registry + tracing.
    fn name(&self) -> &'static str;

    /// Op type(s) this kernel handles.
    fn op_types(&self) -> &[&'static str];

    /// Execute. All inputs/outputs are pre-allocated TensorViews.
    fn execute(&self, ctx: &mut KernelContext) -> Result<()>;

    /// Can this kernel handle the given node? (dtype/attribute check)
    fn can_handle(&self, node: &NodeView) -> bool;

    /// Estimated FLOPs for cost model (optional).
    fn estimate_flops(&self, shapes: &[&Shape]) -> Option<u64> { None }
}

/// Registration macro — one line per kernel.
macro_rules! register_kernels {
    ($registry:expr, $($kernel:ty),* $(,)?) => {
        $( $registry.register(Box::new(<$kernel>::new())); )*
    };
}
```

**Adding a new kernel = one file + one register line.** No touching core runtime code.

### 6.9 Sequence ops: copy-free & race-free container semantics

ONNX `Sequence*` ops operate on a *sequence-of-tensors* container value, which a
tensor-in/tensor-out `Kernel` cannot represent. They are therefore handled
directly by the session executor (like `If`/`Loop`/`Scan`), routed by
`is_sequence_op` — **no EP kernel-registry entry**, so they never collide with
tensor-kernel registration.

**No copy.** A sequence stores its elements as `Arc`-shared **immutable**
tensors (`SeqTensor`). Every value-semantic op returns a *new* list that
**shares the surviving element `Arc`s** with the source (persistent-data-
structure style) — only handles (a refcount bump) are cloned, never element
bytes:

- `SequenceConstruct` shares the input element `Arc`s.
- `SequenceInsert` / `SequenceErase` build a new `Vec<Arc<SeqTensor>>` that
  shares every unchanged element with the input.
- `SequenceAt` returns the **shared element `Arc`** and backs its output tensor
  value with that same allocation (`seq_elem_values`), so a downstream kernel
  reads it through a zero-copy `TensorView`; bytes are materialized only at the
  graph-output boundary.

The only unavoidable copies are boundary crossings: a *tensor → sequence* entry
(a produced `DeviceBuffer`, reused across runs, cannot be aliased, so its bytes
are moved into the element `Arc` exactly once) and the single-alloc data
movement of `SplitToSequence` / `ConcatFromSequence` (`ConcatFromSequence`
necessarily allocates its output once and memcpies each element in exactly
once). This contrasts with stock ORT, whose `SequenceInsert`/`Erase`/`At`
deep-copy element data on every mutation.

**No race.** `SeqTensor` is immutable after construction and shared read-only
through `Arc` (no interior mutability), so concurrent readers of a shared
sequence/element cannot race — the only cross-thread state is `Arc`'s atomic
refcount. `SequenceValue: Send + Sync` is asserted by a unit test, and a
concurrency smoke test runs many threads reading one shared sequence.

The proof that no deep copy occurs is a unit test asserting `Arc::ptr_eq` (and
data-pointer equality) between an element inserted into a sequence and the same
element returned by `SequenceAt` after intervening `insert`/`erase`.



Every operation in the runtime emits structured traces via the `tracing` crate.
**Always compiled in** (no feature gate). Tracing instrumentation is zero-cost when no
subscriber is active — the `tracing` crate compiles spans to no-ops without a collector.

Users opt-in to **collecting** traces at runtime, not at compile time.

### Profiling API (per-language idioms)

**Rust (RAII guard):**
```rust
let session = InferenceSession::load("model.onnx")?;

// Option 1: session-level toggle
let session = InferenceSession::builder()
    .model("model.onnx")
    .profiling(true)  // enable collection
    .build()?;
let data = session.profiling_data();  // after runs
session.export_profile("/tmp/trace.json")?;

// Option 2: scoped profiling (like PyTorch)
{
    let _guard = nxrt::profiler::record("/tmp/trace.json");
    // everything in this scope is profiled
    session.run(&inputs)?;
    session.run(&inputs)?;
}  // guard drops → flushes trace file

// Option 3: global subscriber (for library integrators)
nxrt::profiler::install_global(ChromeTraceSubscriber::new("/tmp/trace.json"));
```

**Python (context manager, like `torch.profiler.profile`):**
```python
import nxrt

session = nxrt.load("model.onnx")

# Context manager — idiomatic Python
with nxrt.profiler.profile(output="/tmp/trace.json") as prof:
    output = session.run(input_ids=data)
    output = session.run(input_ids=data)
# Exiting context → trace flushed to file

# Access programmatically
with nxrt.profiler.profile() as prof:
    session.run(input_ids=data)
print(prof.summary())           # human-readable table
prof.export_chrome_trace("trace.json")
prof.export_perfetto("trace.perfetto")

# Or decorate a function
@nxrt.profiler.trace
def generate(session, prompt):
    # ... all ops inside are traced
    pass

# Session-level (always-on for this session)
session = nxrt.load("model.onnx", profiling=True)
session.run(input_ids=data)
print(session.profiling_summary())
```

**C API (start/stop pattern):**
```c
// C: explicit start/stop (matches ORT's existing pattern)
OrtStatus* status;
status = OrtSessionStartProfiling(session, "/tmp/trace.json");
// ... run inference ...
status = OrtSessionStopProfiling(session);  // flushes

// Or: get profile data as string
char* profile_json = NULL;
status = OrtSessionGetProfilingData(session, &profile_json);
```

**CLI:**
```bash
# Command-line profiling (wraps any model run)
nxrt profile model.onnx --input data.npz --output trace.json --runs 10
nxrt profile model.onnx --input data.npz --summary  # prints table to stdout
```

**Cost model uses real traces for auto-tuning:**
```rust
impl CostModel {
    /// Update cost estimates from actual execution traces.
    /// Called after warmup runs to replace heuristic estimates with measured data.
    pub fn calibrate_from_traces(&mut self, traces: &[TraceEvent]) {
        for event in traces.iter().filter(|e| e.name == "kernel.execute") {
            self.update_kernel_cost(event.op_type, event.device, event.duration_us);
        }
    }
}
```

## 8. Custom Ops Extension Point

Users can register additional ops via the C ABI custom op interface (same mechanism as ORT Extensions):

```rust
pub trait CustomOp: Send + Sync {
    fn name(&self) -> &str;
    fn domain(&self) -> &str;
    fn since_version(&self) -> u64;
    fn create_kernel(&self, node: &Node) -> Result<Box<dyn Kernel>>;
}

impl InferenceSession {
    /// Register custom ops from a shared library.
    pub fn register_custom_ops(path: &Path) -> Result<()>;
}
```
