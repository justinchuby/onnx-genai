## 2026-07-23 — CUDA IndexShare (GLM/DeepSeek DSA) + f16 standard Attention

- **f16 CUDA standard Attention ✅ (`07e4c80`, Roy):** the CUDA standard `Attention` op — previously **f32-only** — now accepts homogeneous f32 **or f16** Q/K/V, paired KV-cache tensors, and f16 additive masks (incl. `-inf`/`-65504`), writing `Y`, present KV, and the optional QK output in the selected dtype. All score/softmax/value reductions retain **fp32 accumulation** (reusing the f16 GQA load/store + fp32-accumulate pattern). bf16 stays rejected as a follow-up. This closes the GLM/DeepSeek "checked floating inputs of Attention/RoPE remain f32-only" dtype-coverage gap in `docs/GLM_READINESS_GAPS.md` and halves activation/KV bandwidth for real fp16 exports. The parity test `standard_attention_fp16_gpu.rs` was hardened after review (Gaff 🟡→addressed): exact f16-rounded CPU-f32 oracle, seq=32/head=64/multi-head, checks Y+present-KV+QK, tolerance tightened to `3e-4`, and a **mutation check proves the guard** (forcing f16 score accumulation drives QK error to `3.996e-4` > tolerance → test FAILS). Merged max|Δ|: prefill `Y` 1.94e-4, decode `Y` 2.99e-5, caches bit-exact.
- **Device-resident CUDA IndexShare v1 ✅ (`1304707`+`0828abb`, Keaton):** new GPU-native kernel for `pkg.nxrt::IndexShare` v1 — GLM-5.2 IndexShare / DeepSeek DSA **selected-token attention** — which previously had only a CPU reference. Two NVRTC kernels: `build_present` (device `past⧺current` KV concat, bit-identical to the CPU concat) and `index_share_row` (one block per `(batch, q_head, query)`: device selected-token gather, scaled QK, optional additive bias, numerically-stable **fp32** softmax, prob·value sum). Q/K/V/bias/present-cache/output stay device-resident; only the small `selected_indices` tensor round-trips to the host for the deterministic ONNX index validation (SparseKvGather precedent), so `cuda_graph_compatible()==false` is explicitly gated. The claim gate **delegates to the CPU oracle's** `unsupported_reason` (made `pub`) so both backends reject identical combos. Reviewed by Chew (🟡→addressed; independently re-ran all tests green on H200, traced numerics/indexing/GQA-grouping/KV-threading/memory-safety line-by-line): fixed a rank-0 scalar `attention_bias` claim-then-fail divergence (CPU accepted it, CUDA rejected at execution) by accepting rank-0 as a broadcast scalar, with a bit-exact `scalar_bias_broadcasts_and_matches_cpu` guard. Parity: present KV bit-identical, `Y` max|Δ| ≤ **2.38e-7** (1e-4 tolerance). **7 GPU parity tests** (incl. prefill→decode→decode threading present_*→past_*).
- **Integration verified on merged `main`:** `index_share_gpu` 7/7, `standard_attention_fp16_gpu` 1/1, `standard_attention_gpu` 23/23; `cargo clippy -p onnx-runtime-ep-cuda --features cuda -- -D warnings` clean. Both kernels model-agnostic (no model-name dispatch, RULES §2). Remaining GLM/DeepSeek follow-ups: IndexShare CUDA-graph capture (device-side index validation), f16/bf16 IndexShare storage variants, bf16 standard Attention, Mobius fused QMoE/BlockQuantizedMoE emitter, MTP state threading.

## Design → Implementation Gap Analysis (2026-07-14, Tyrell)

Evidence-based audit of `docs/DESIGN.md` (engine) + `docs/ORT2.md` (nxrt runtime) against the actual workspace crates. Verified by inspecting registrations/types/tests — **not** by trusting the prose above. Legend: ✅ implemented + tested · 🟡 partial/stubbed · ❌ design-only / not started. Ordered roughly by roadmap impact.

### Runtime (nxrt) — top gaps

1. **Op coverage (ep-cpu) — 🟡 the #1 conformance blocker.** `onnx-runtime-ep-cpu` registers only **~30 op types** (24 standard `ai.onnx`: Add, Sub, Mul, Div, Pow, Min, Max, Sqrt, Erf, Tanh, MatMul, Gemm, Relu, LayerNormalization, ReduceMean, Softmax, Reshape, Transpose, Gather, Cast, Shape, Slice, Unsqueeze, Expand, Constant, Identity + 6 fused/contrib: FusedMatMulBias, FusedGemm, FusedAttention, Gelu) — see `crates/onnx-runtime-ep-cpu/src/kernels/mod.rs`. The upstream ONNX opset is ~200 ops; **~90+ core ops are unregistered** (Conv, MaxPool/AveragePool/GlobalPool, Sigmoid, Abs, Clip, Concat, Split, Where, Squeeze, Pad, Sigmoid/GELU-variants, ArgMax, TopK, all recurrent RNN/GRU/LSTM, all quantization Q*/DequantizeLinear/MatMulInteger, Resize, etc.). Upstream conformance (`docs/EP_CONFORMANCE.md`): **158 / 1198** dtype-opset pytest cases pass (1038 fail, 2 skip); the Rust runner covers 25 PASS / 3 UNSUPPORTED. **This gap gates real-model coverage and the whole conformance story.**

2. **Kernel dtype coverage (ep-cpu) — 🟡 f32-only compute.** `kernels/mod.rs::to_dense_f32` calls `require_dtype(..., Float32)` — every compute kernel rejects non-f32. Only `Identity` handles bf16/f16/i32 bit-exact (`kernels/identity.rs`). **No fp16/bf16/int8 compute kernels exist**, so quantized and half-precision models cannot run on the CPU EP even for registered ops. (IR/loader/`Value` model bf16/f16/fp8/int4 correctly — the gap is purely in kernels.) Unblocks quantized-model milestone alongside op coverage.

3. **CUDA EP Phase 2b — 🟡 minimal GPU slice.** `onnx-runtime-ep-cuda` registers only **`MatMul` (cuBLASLt, true fp32/f16/bf16) + a naive `Attention` SDPA baseline** (`kernels/mod.rs::CUDA_PHASE2A_OPS`); the SDPA kernel is **f32-only** and explicitly *not* FlashAttention (`kernels/attention.rs`). Deferred (clear `// Phase 2b:` markers): cuDNN-fused SDPA → **FlashAttention-3**, fused LayerNorm/RMSNorm/RoPE/softmax via CuTe, NVRTC elementwise, **FP8**, fused bias/act epilogues, strided/broadcast batched GEMM, per-stream workspace pooling, true async-copy fences, **CUDA-graph capture**, and **paged-KV §13.4**. Blocks H200 perf and device-resident serving.

4. **Tracer executor wiring / MissedFastPath real emission sites — 🟡 seams only.** `onnx-runtime-tracer` (Chrome/Perfetto export, AutoDiagnosis §46.6, Roofline §49.5, `IssueCategory::MissedFastPath`, CUPTI/ITT collectors) is fully built **but not imported by any runtime crate** — `grep onnx-runtime-tracer crates/*/Cargo.toml` matches only the tracer's own files. No `Executor`/session instrumentation, no §46 execution-trace log, no `tracing`-crate `Layer` bridge (§17.1). The `MissedFastPath`/Roofline diagnoses run only on synthetic test spans; **there are no real kernel emission sites feeding them.**

5. **Optimizer passes beyond fusion — 🟡/❌.** `onnx-runtime-optimizer` ships **ConstantFolding ✅ / DeadNodeElimination ✅ / OpFusion ✅** (LayerNorm, FusedMatMulBias, FusedGemm, FusedAttention, Gelu — all with CPU kernels + shape rules + bert_toy parity). **`LayoutPropagation`, `PlacementOptimizer`, and cost-model-driven device placement are design-only** — named but commented out in `optimizer/src/lib.rs::default_passes()`.

6. **Eager execution (`docs/EAGER.md`) — 🟡 core dispatch only.** `onnx-runtime-eager` implements real single-op dispatch by building ephemeral IR nodes, resolving kernels through the ep-cpu `ExecutionProvider` and shapes through `onnx-runtime-shape-inference` (`src/dispatch.rs`). It is therefore **bounded by ep-cpu's ~30 ops / f32-only** limits, the §9.2 kernel-shape fallback is deferred, it is a **standalone crate not wired into any Python surface**, and custom-domain / subgraph / autograd features are unbuilt.

7. **EPContext §55 — ✅ COMPLETE (recorded for accuracy).** Produce → dump (`*_ctx.onnx`, embed/external) → reload → consume + `ep.context_*` C-API options are all landed and model-agnostic (`onnx-runtime-loader/src/{epcontext,writer,encoder}.rs`, `session/src/epcontext.rs`, capi). No CPU-EP compile step exercises it yet (mock-EP tested); real auto-export awaits a compiling EP.

### Engine (onnx-genai) — memory / serving layer (the largest design-only surface)

8. **`KvCacheManager` abstraction — ❌ not realized as designed.** DESIGN §3.2/§decode-loop centre everything on a `KvCacheManager` (page-table mapping + append). **No such type exists** (`grep KvCacheManager crates/` → nothing); the engine composes `PageTable` + `PrefixCache` + `PagedCache` (`onnx-genai-kv`) directly. Functionally the KV path works, but the designed manager seam that the tiered store / budget / governor plug into is absent — which is *why* items 9–11 are hard to add.

9. **Tiered storage §3.2.3 (GPU↔CPU↔SSD) — 🟡 no SSD tier.** GPU↔CPU hot/cold tiering is implemented (`LocalTieredConnector` + `PageTable` LRU offload, `onnx-genai-kv/src/local_tiered.rs`). The **SSD/disk cold tier is a config-only stub** — the code comments state "*This milestone does not implement a real disk spill*"; a mmap/direct-I/O `LocalDisk` backend is unbuilt. §38 distributed connector trait ✅, but the only real backend is local GPU↔CPU.

10. **MemoryBudget & cross-session eviction §26.4 — 🟡 shared byte-budget primitive and engine-governor wiring landed.** `onnx-genai-scheduler` has admission, priority classes, FCFS/Priority policies, and a real deficit-weighted FairShare policy with configurable class weights, bounded anti-starvation, and idle-credit reset; it also has a per-scheduler `max_total_tokens` KV budget and preempt-evict-to-CPU (`scheduler/src/lib.rs`). The **global cross-session dynamic byte-budget accounting** (bytes, not tokens; shared across sessions/models) now exists as `scheduler::byte_budget::ByteBudget` — a cloneable, thread-safe, dynamically-reconfigurable byte ceiling. `Scheduler::with_byte_budget` gates admission/swap-in on it (bytes derived from a model-supplied `SchedulerConfig.bytes_per_token`, RULES #2), releasing on completion/preemption; over-budget rejections carry the RULES #1 what/why/how shortfall. Two schedulers sharing one handle account against one device ceiling (DESIGN §26.11.3). Engine configuration and guarded live limit updates are now wired; **live eviction on a lowered limit remains pending**.

11. **Resource Governor §26.11 — 🟡 engine wiring complete; live eviction deferred.** The engine now parses byte/fraction/`auto` limits, reads `EngineConfig.limits` and `allow_runtime_override` from YAML, exposes governor and per-tier resource snapshots, and guards runtime VRAM-limit updates (`1eebf5d`). The §26.11 design (`d6736e1`) remains the source for transactional lowering and actionable errors; immediate eviction to satisfy a lowered live limit is not yet implemented.
