## Design ŌåÆ Implementation Gap Analysis (2026-07-14, Tyrell)

Evidence-based audit of `docs/DESIGN.md` (engine) + `docs/ORT2.md` (nxrt runtime) against the actual workspace crates. Verified by inspecting registrations/types/tests ŌĆö **not** by trusting the prose above. Legend: Ō£ģ implemented + tested ┬Ę ­¤¤Ī partial/stubbed ┬Ę ŌØī design-only / not started. Ordered roughly by roadmap impact.

### Runtime (nxrt) ŌĆö top gaps

1. **Op coverage (ep-cpu) ŌĆö ­¤¤Ī the #1 conformance blocker.** `onnx-runtime-ep-cpu` registers only **~30 op types** (24 standard `ai.onnx`: Add, Sub, Mul, Div, Pow, Min, Max, Sqrt, Erf, Tanh, MatMul, Gemm, Relu, LayerNormalization, ReduceMean, Softmax, Reshape, Transpose, Gather, Cast, Shape, Slice, Unsqueeze, Expand, Constant, Identity + 6 fused/contrib: FusedMatMulBias, FusedGemm, FusedAttention, Gelu) ŌĆö see `crates/onnx-runtime-ep-cpu/src/kernels/mod.rs`. The upstream ONNX opset is ~200 ops; **~90+ core ops are unregistered** (Conv, MaxPool/AveragePool/GlobalPool, Sigmoid, Abs, Clip, Concat, Split, Where, Squeeze, Pad, Sigmoid/GELU-variants, ArgMax, TopK, all recurrent RNN/GRU/LSTM, all quantization Q*/DequantizeLinear/MatMulInteger, Resize, etc.). Upstream conformance (`docs/EP_CONFORMANCE.md`): **158 / 1198** dtype-opset pytest cases pass (1038 fail, 2 skip); the Rust runner covers 25 PASS / 3 UNSUPPORTED. **This gap gates real-model coverage and the whole conformance story.**

2. **Kernel dtype coverage (ep-cpu) ŌĆö ­¤¤Ī f32-only compute.** `kernels/mod.rs::to_dense_f32` calls `require_dtype(..., Float32)` ŌĆö every compute kernel rejects non-f32. Only `Identity` handles bf16/f16/i32 bit-exact (`kernels/identity.rs`). **No fp16/bf16/int8 compute kernels exist**, so quantized and half-precision models cannot run on the CPU EP even for registered ops. (IR/loader/`Value` model bf16/f16/fp8/int4 correctly ŌĆö the gap is purely in kernels.) Unblocks quantized-model milestone alongside op coverage.

3. **CUDA EP Phase 2b ŌĆö ­¤¤Ī minimal GPU slice.** `onnx-runtime-ep-cuda` registers only **`MatMul` (cuBLASLt, true fp32/f16/bf16) + a naive `Attention` SDPA baseline** (`kernels/mod.rs::CUDA_PHASE2A_OPS`); the SDPA kernel is **f32-only** and explicitly *not* FlashAttention (`kernels/attention.rs`). Deferred (clear `// Phase 2b:` markers): cuDNN-fused SDPA ŌåÆ **FlashAttention-3**, fused LayerNorm/RMSNorm/RoPE/softmax via CuTe, NVRTC elementwise, **FP8**, fused bias/act epilogues, strided/broadcast batched GEMM, per-stream workspace pooling, true async-copy fences, **CUDA-graph capture**, and **paged-KV ┬¦13.4**. Blocks H200 perf and device-resident serving.

4. **Tracer executor wiring / MissedFastPath real emission sites ŌĆö ­¤¤Ī seams only.** `onnx-runtime-tracer` (Chrome/Perfetto export, AutoDiagnosis ┬¦46.6, Roofline ┬¦49.5, `IssueCategory::MissedFastPath`, CUPTI/ITT collectors) is fully built **but not imported by any runtime crate** ŌĆö `grep onnx-runtime-tracer crates/*/Cargo.toml` matches only the tracer's own files. No `Executor`/session instrumentation, no ┬¦46 execution-trace log, no `tracing`-crate `Layer` bridge (┬¦17.1). The `MissedFastPath`/Roofline diagnoses run only on synthetic test spans; **there are no real kernel emission sites feeding them.**

5. **Optimizer passes beyond fusion ŌĆö ­¤¤Ī/ŌØī.** `onnx-runtime-optimizer` ships **ConstantFolding Ō£ģ / DeadNodeElimination Ō£ģ / OpFusion Ō£ģ** (LayerNorm, FusedMatMulBias, FusedGemm, FusedAttention, Gelu ŌĆö all with CPU kernels + shape rules + bert_toy parity). **`LayoutPropagation`, `PlacementOptimizer`, and cost-model-driven device placement are design-only** ŌĆö named but commented out in `optimizer/src/lib.rs::default_passes()`.

6. **Eager execution (`docs/EAGER.md`) ŌĆö ­¤¤Ī core dispatch only.** `onnx-runtime-eager` implements real single-op dispatch by building ephemeral IR nodes, resolving kernels through the ep-cpu `ExecutionProvider` and shapes through `onnx-runtime-shape-inference` (`src/dispatch.rs`). It is therefore **bounded by ep-cpu's ~30 ops / f32-only** limits, the ┬¦9.2 kernel-shape fallback is deferred, it is a **standalone crate not wired into any Python surface**, and custom-domain / subgraph / autograd features are unbuilt.

7. **EPContext ┬¦55 ŌĆö Ō£ģ COMPLETE (recorded for accuracy).** Produce ŌåÆ dump (`*_ctx.onnx`, embed/external) ŌåÆ reload ŌåÆ consume + `ep.context_*` C-API options are all landed and model-agnostic (`onnx-runtime-loader/src/{epcontext,writer,encoder}.rs`, `session/src/epcontext.rs`, capi). ### Engine (onnx-genai) ŌĆö memory / serving layer (the largest design-only surface)

8. **`KvCacheManager` abstraction ŌĆö ŌØī not realized as designed.** DESIGN ┬¦3.2/┬¦decode-loop centre everything on a `KvCacheManager` (page-table mapping + append). **No such type exists** (`grep KvCacheManager crates/` ŌåÆ nothing); the engine composes `PageTable` + `PrefixCache` + `PagedCache` (`onnx-genai-kv`) directly. 9. **Tiered storage ┬¦3.2.3 (GPUŌåöCPUŌåöSSD) ŌĆö ­¤¤Ī no SSD tier.** GPUŌåöCPU hot/cold tiering is implemented (`LocalTieredConnector` + `PageTable` LRU offload, `onnx-genai-kv/src/local_tiered.rs`). 10. **MemoryBudget & cross-session eviction ┬¦26.4 ŌĆö ­¤¤Ī shared byte-budget primitive and engine-governor wiring landed.** `onnx-genai-scheduler` has admission, priority classes, FCFS/Priority policies, and a real deficit-weighted FairShare policy with configurable class weights, bounded anti-starvation, and idle-credit reset; it also has a per-scheduler `max_total_tokens` KV budget and preempt-evict-to-CPU (`scheduler/src/lib.rs`). The **global cross-session dynamic byte-budget accounting** (bytes, not tokens; shared across sessions/models) now exists as `scheduler::byte_budget::ByteBudget` ŌĆö a cloneable, thread-safe, dynamically-reconfigurable byte ceiling. `Scheduler::with_byte_budget` gates admission/swap-in on it (bytes derived from a model-supplied `SchedulerConfig.bytes_per_token`, RULES #2), releasing on completion/preemption; over-budget rejections carry the RULES #1 what/why/how shortfall. Engine configuration and guarded live limit updates are now wired; **live eviction on a lowered limit remains pending**.

11. **Resource Governor ┬¦26.11 ŌĆö ­¤¤Ī engine wiring complete; live eviction deferred.** The engine now parses byte/fraction/`auto` limits, reads `EngineConfig.limits` and `allow_runtime_override` from YAML, exposes governor and per-tier resource snapshots, and guards runtime VRAM-limit updates (`1eebf5d`). The ┬¦26.11 design (`d6736e1`) remains the source for transactional lowering and actionable errors; immediate eviction to satisfy a lowered live limit is not yet implemented.

12. **Multimodal / VLM & diffusion ŌĆö ­¤¤Ī/ŌØī.** Image + audio **preprocessing** Ō£ģ (`onnx-genai-preprocess`: bicubic/CLIP + tiling + log-mel), single-tensor `pixel_values` vision path Ō£ģ, audio input wired Ō£ģ. 13. **Speculative decoding ŌĆö Ō£ģ correctness, ­¤¤Ī perf.** Draft + prompt-lookup + MTP + EAGLE-3 + Gemma4-assistant (shared-KV) proposers are implemented and **token-identical to greedy on real Gemma4 E2B weights on H200** (┬¦27).
