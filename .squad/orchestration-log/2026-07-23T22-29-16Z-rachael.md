# Rachael — DeepSeek MLA garbage root cause and fix

- Root-caused real DeepSeek-V2-Lite native CUDA garbage decode to a device-stream ordering race in `CudaRuntime::dtod`.
- `cuMemcpyDtoD` ran on the legacy default stream, unordered against the EP non-blocking compute stream; `copy_reshape` could read stale `k_nope` and V columns in the MLA K/V chain.
- Fixed on `fix/deepseek-mla-attention-prefill` @ `1fe314f` by synchronizing before `memcpy_dtod_sync`, mirroring `dtoh`.
- Added regression `runtime::tests::dtod_waits_for_pending_stream_writes`.
