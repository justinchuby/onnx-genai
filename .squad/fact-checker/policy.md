# Fact Checker Policy

## Verification scope
- External references (URLs, crate names, versions on crates.io).
- API signatures (ORT, tokenizers, std) claimed in code or docs.
- Conformance claims against the ONNX Inference Metadata Standard.

## Devil's Advocate triggers
- Significant architecture or design decisions.
- Risky launches or public API commitments.
- When the team converges too fast without dissent.

## Rules
- Trust, but verify. Then steelman the opposition.
- Advisory by default; escalate only provably false claims or unaccepted risks.
